## Design Philosophy

- **Data-driven**: Surface insights from synced Strava activities
- **Goal-oriented**: Focus on training plans and progress tracking
- **Clean & functional**: Prioritize usability over visual complexity
- **Mobile-first**: Optimize for athletes checking plans on-the-go

### Data-Driven Layout Considerations

Given the rich activity data available, the layout specification prioritizes:

1. **Visual Charts**: Leverage `streams` data for pace charts, HR zones, elevation profiles
2. **Lap Analysis**: Display detailed `laps[]` breakdowns in activity detail views
3. **Segment Performance**: Show `segmentEfforts[]` with PR rankings and achievements
4. **Split Comparisons**: Use `splitsMetric[]` and `splitsStandard[]` for consistency analysis
5. **Heart Rate Zones**: Calculate zone distribution from HR stream data
6. **Training Load**: Compute metrics like TSS from power/HR data when available

## Responsive Breakpoints

### Mobile (<768px)

- Bottom navigation (fixed)
- Single column layouts
- Collapsible sections
- Swipeable cards
- Touch-optimized controls

### Tablet (768px - 1023px)

- Side navigation or top tabs
- Two-column layouts where appropriate
- Larger touch targets
- Responsive charts

### Desktop (â‰¥1024px)

- Persistent sidebar navigation
- Multi-column layouts
- Hover states and tooltips
- Detailed visualizations
- Max width: 1400px (centered)

### Component Patterns

**Cards**

- White background (light mode) / Gray-800 (dark mode)
- Border radius: 12px
- Shadow: sm (default), md (hover), lg (elevated)
- Padding: 16px (mobile), 24px (desktop)

**Buttons**

- Primary: Filled with primary color
- Secondary: Outlined
- Ghost: Text only
- Sizes: sm (32px), md (40px), lg (48px)
- Border radius: 8px

**Inputs**

- Height: 40px (md), 48px (lg)
- Border: 1px solid gray-300
- Border radius: 8px
- Focus: 2px ring, primary color

**Charts**

- Clean, minimal design
- Primary color for main data
- Gray tones for secondary data
- Tooltips on hover
- Responsive sizing

## Loading & Empty States

### Loading States

- Skeleton screens for cards and content
- Spinner for button actions
- Progress bars for long operations (sync)
- Shimmer animation

### Empty States

- Friendly illustrations
- Clear message
- Call-to-action button
- Example: "No activities yet. Complete your first workout!"

### Error States

- Error icon
- Clear error message
- Suggested action
- Retry button

## Accessibility

- Semantic HTML elements
- ARIA labels where needed
- Keyboard navigation support
- Focus indicators
- Screen reader announcements
- Color contrast (WCAG AA minimum)
- Alt text for images
- Proper heading hierarchy

## Performance

- Lazy load routes
- Virtualize long lists (activities)
- Optimize images
- Cache API responses
- Debounce search/filters
- Progressive loading of charts
- Service worker for offline support
